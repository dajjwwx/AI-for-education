# AI教育讲座扫码答题系统 使用说明

## 快速启动

1. 双击 `start.bat` 启动服务器
2. 浏览器会自动打开讲座课件
3. 在另一个浏览器标签打开 `http://localhost:3000/dashboard` 查看数据看板

## 使用流程

### 讲座前准备
1. 确保电脑和手机连接同一个WiFi网络
2. 双击 `start.bat` 启动系统
3. 讲座课件会自动在浏览器打开
4. 另开一个标签页打开数据看板：`http://localhost:3000/dashboard`

### 讲座中操作
1. 正常演示讲座课件（按方向键翻页）
2. 每个部分讲完后，会出现一张「扫码答题」幻灯片
3. 让老师们用手机微信扫码进入答题页面
4. 答题幻灯片右侧会实时显示答题统计（每3秒自动刷新）
5. 也可以在数据看板页面查看详细的实时分析

### 答题环节说明
| 环节 | 位置 | 题目数 | 内容 |
|------|------|--------|------|
| 第1部分 | 开场后 | 5题 | AI来了教育不能装睡 |
| 第2部分 | 六大能力后 | 5题 | AI时代稀缺的六大能力 |
| 第3部分 | 教师角色后 | 5题 | 教师角色再定位 |
| 第4部分 | AI赋能后 | 5题 | AI赋能教学 |
| 第5部分 | 教学方法后 | 5题 | 课堂教学方法 |
| 第6部分 | 结语后 | 5题 | 做有温度的教育 |

共30道题，每环节约2-3分钟。

### 老师扫码答题流程
1. 手机扫描课件上的二维码
2. 输入姓名（可选）→ 开始答题
3. 逐题选择答案 → 提交
4. 立即看到：个人得分、每题对错、错题解析、错误原因分析、学习推荐

### 数据看板功能
- 实时答题人数、平均分、最高分、最低分
- 分数段分布柱状图
- 逐题正确率进度条
- 常见错误选项分布
- 薄弱环节分析（正确率<60%的题目）
- 个性化学习推荐
- 最近提交记录

## 文件结构

```
quiz-server/
├── start.bat           一键启动脚本
├── server.js            服务器主程序
├── questions.json       30道题目数据
├── package.json         依赖配置
├── data.json            答题数据（自动生成，可删除重置）
└── public/
    ├── slides.html      讲座课件（含扫码答题幻灯片）
    ├── quiz.html        手机端答题页面
    └── dashboard.html   讲师端数据看板
```

## 注意事项

1. **网络要求**：手机和电脑需在同一WiFi下，手机才能访问答题页面
2. **防火墙**：如Windows防火墙拦截了访问，请允许Node.js通过防火墙
3. **重置数据**：删除 `data.json` 文件即可清空所有答题记录
4. **IP地址**：启动时控制台会显示本机IP，QR码会自动使用该IP
5. **端口**：默认使用3000端口，如被占用请修改 `server.js` 中的 PORT

## 技术栈

- 后端：Node.js + Express
- 数据存储：JSON文件
- QR码：qrcode npm包
- 前端：原生HTML/CSS/JS
- 实时更新：AJAX轮询（3秒间隔）
